The CSS Typography classes were designed with specific values for font-family, font-weight, font-size, line-height, etc. and may not meet the needs of all applications. The following demonstrates how to override these CSS properties while ensuring the multi-language font fallback list is still provided.

Start by applying a CSS typographic class on an element, then create another class that overrides CSS font properties.

Note if the font-family property is overriden, font fallback will happen from the system level rather than from ui-light.css/ui-dark.css.

Ensure that the application links in ui-light.css or ui-dark.css files for multi-language font fallback support (Note: This sample links to ui-light.css by default).
