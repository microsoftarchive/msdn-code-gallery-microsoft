This scenario demonstrates how to display grouped data using custom group and item datasources. Binding.List can be used to provide simple sorting and grouping for in-memory data, but does not scale to large datasets. Using a custom group & item data sources enables listview to scale to large datasets of grouped data by using data virtualization. There are seperate datasources for the items and the groups, both implemented using custom data adapters, wrapped with a VirtualizedDataSource object.

The item data adapter needs to supply the items sorted by group. It also must provide a groupKey for each item to specify the group.

The group data adapter supplies the list of groups in the same order as the item collection. It must provide a firstItemIndexHint for each group to indicate where the group starts. Groups can be enumerated by key or index, so it implements both itemsFromIndex and itemsFromKey.
