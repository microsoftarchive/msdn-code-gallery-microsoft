The WinJS Tooltip control has two modes: normal tooltip and infotip. Use the normal tooltip mode for short words; this mode is helpful for assisting with the finger occlusion problem that occurs when the user interacts with the control via a touch interface. Use the infotip mode when you want to display content that has images or more than a sentence of text. The infotip requires a longer holding time before it appears.

You could provide any DOM element as content, even with WinJS controls inside. The tooltip control will re-parent the element to the tooltip container, and block interaction events on that element, since that's not the suggested interaction model.
