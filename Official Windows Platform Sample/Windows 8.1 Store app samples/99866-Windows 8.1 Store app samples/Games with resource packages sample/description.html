This sample demonstrates resource packages used for games.

Larger Windows Store games, especially those that support multiple languages with region-specific assets or feature optional high-definition assets, can easily balloon to large sizes. Resource packs let you provide an app experience where your customers only receive the resources they actually need.

Resource packs contain optional or expanded platform-agnostic data, such as game assets (textures, meshes, sound, text). A Windows Store game can have one or more resource packs, including resource packs for high-definition assets or textures, Direct feature level 11_0+ resources, or language-specific assets and resources.

Specifically, this sample provides code for:

  • Surface-shaded lighting of an object using a normal map and a color texture.
  • Very basic pinch-to-zoom controls implemented without the gesture recognizer.

This sample is written in C++ and requires some experience with graphics programming and DirectX.

Build the sample

  1. Start Visual Studio 2013 and select File > Open > Project/Solution.
  2. Go to the directory in which you unzipped the sample. Go to the directory named for the sample, and double-click the Visual Studio 2013 Solution (.sln) file.
  3. Press F7 or use Build > Build Solution to build the sample.

Run the sample

To debug the app and then run it, press F5 or use Debug > Start Debugging. To run the app without debugging, press Ctrl+F5 or use Debug > Start Without Debugging.
