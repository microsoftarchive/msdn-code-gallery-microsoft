The default style sheets ui-light.css and ui-dark.css provide a series of CSS font fallback definitions to support multiple languages within an application. This is helpful when an application hosts content that is a different language than the default language of the application.

  1. If the characters are not covered by the CSS font fallback list the system will render the glyphs using the system provided font fallback list.
  2. If the characters are covered by a font in the CSS font fallback list, the first font that contains the character will be used.
  3. If the application knows what language the characters are from and uses a CSS typopgrahpic style, the first font associated with that language will be used.

Ensure that the application links in ui-light.css or ui-dark.css files for multi-language font fallback support (Note: This sample links to ui-light.css by default).
