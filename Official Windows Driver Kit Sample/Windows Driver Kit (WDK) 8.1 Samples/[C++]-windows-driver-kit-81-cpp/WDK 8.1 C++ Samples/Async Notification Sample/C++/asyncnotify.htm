AsyncNotify Sample

 

Summary

This sample demonstrates how to implement AsyncNotification to communicate between registered applications and printing components loaded in the spooler. The client part of the sample is a simple application that registers for Async print notifications and acts as the listener. The server part of the sample opens a notification channel and acts as the sender. The server part of the sample is compiled as a library to be linked to a port monitor or a print processor.

 

Note: This sample has not been thoroughly tested and is provided here only as a proof of concept.

Client sample (ddkasyncnotify.exe)

The main function of the client sample is to register for Async notifications and act as a listener for any notifications being sent by print components loaded by the spooler. The callback module implemented in acallback.cpp creates an object exposing the IPrintAsyncNotifyCallback interface. The notifydata module is provided to implement an object that exposes the IPrintAsyncNotifyDataObject interface. The notification schema used in this sample is identified by SAMPLE_NOTIFICATION_UI and is intended only for the purposes

 

Server sample (ServerSampleLib.lib)

This sample is provided only to illustrate how two different print components loaded by the spooler can communicate to listening clients via Async notifications. The sample ilistrates sending both notifications with a private schema and Microsoft UI proprietary schema.

 

  • The changes relevant to Async Notification are as follows:

    • o        The sample creates an object exposing the IPrintAsyncNotifyCallback interface. This code is in acallback.cpp and acallback.hpp.

    o        The sample creates an object exposing the IPrintAsyncNotifyDataObject interface. This code is in notifydata.cpp and notifydata.hpp.

    o        The sample creates an Async notification channel and sends a notification data object packaged in an IPrintAsyncNotifyDataObject object. This code is in the SendAsyncNotification function in asyncnotify.cpp.

    o        The sample closes the notification channel after receiving the notification response from the client.

     

    Building the sample

    To build the samples, run “build –cZ” from the directory. The resulting EXE (in the case of the client) or LIB (in the case of the server) will be placed in the appropriate platform directory (i.e., i386 or Alpha). To create a version with verbose debug output, add _DEBUG to the compile defines in the sources file.

     

    Installing the sample

    After building the samples, you can install the different parts of this sample as follows:

    • Client sample: The client sample does not require any installation. The only step necessary is to copy the executable to the target machine you are using for testing. The client application can use the received notifications in any manner it sees fit. The sample ilistrates the most common usage which is showing UI based on the content of the notification. This scenario becomes the only way components running inside the Print Spooler service can show UI in the printing process.
    • ServerSampleLib: To test the server side of the sample, a port monitor or print processor should be developed. These components should link to this sample LIB and invoke the methods in asyncnotify.cpp file to send notifications at different stages of printing. The notifications of type SAMPLE_NOTIFICATION_UI in this sample are only for the purposes of this sample. The notification schema should be designed to fit the purpose of sending them.

       

      Testing the sample

      After installing the sample, you can test the sample as follows:

      • Run ddkasyncnotify.exe from a command prompt window.
      • Initiate a print job through the print driver/print queue that uses the server side component.
      • Calling the method SendAsyncNotification in the server sample from a port monitor or print processor will render the client sample application to show message box.
      • Calling the method SendAsyncUINotification in the server sample from a port monitor or print processor will render the client sample application to show a baloon showing this text: This text is an Async UI sample.
